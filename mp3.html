MP3 Write-Up: Dev Toolchain

1. Overview & Usage:
My project uses the P5 and P5 Play libraries to execute a basic pong game. It uses simple derivatives and built-in functions to create the changing positions of the ball and the opponent paddle.
2. Development Process
• I set up my development environment using NPM.
• Following, I used the P5 editor to visualize the ball and paddles.
• In addition, I also used the editor to experiment with the game logic.
• I continued to make edits, adding components such as the game over screen, restart function, and score tracker.
• After, I used esbuild to bundle my code and Github Pages to deploy it.
3. Issue Deep-Dive
Initially, I had trouble with the ball and paddles correctly working. The program would often show the game over screen on the initial hit of the paddle with the ball. The parameters that dictated when the ball would be considered out of bounds conflicted with its dimensions and position with the paddles. This was due to me initially wanting to trigger the game over screen on the ball passing a certain distance threshold. As a result, the game went through numerous bugs, such as the game over screen triggering on impact with the opponent paddle and conflicts with the active components of the visuals. What worked instead, was making it trigger upon not hitting the paddles instead. Using the simple constrain function allowed me to designate a set area as the “safe” area that would return the ball.
